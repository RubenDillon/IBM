Integracion con Concert
=

Configurar la Integracion
-

1. Crear un subdominio en DuckDNS
DuckDNS te permite registrar un subdominio dinámico gratis.

Ir a https://www.duckdns.org/

Iniciá sesión con tu cuenta GitHub/Google.

Elegí un subdominio, por ejemplo: ibmconcert.duckdns.org

Te va a mostrar un token, por ejemplo: 842c7a3-a5f5-4001-xxxx

Anotá el subdominio y el token, los vas a usar en el script.

2. Script para ejecutar y solicitar el certificado en Lets Encrypt

```
#!/bin/bash

# Configuración
DOMAIN="ibmconcert"
TOKEN="7842c7a3-a5f5-4xxxxxxx"
FULL_DOMAIN="$DOMAIN.duckdns.org"
IP="52.118.151.40"
EMAIL="tu-email@example.com"  # Reemplazá con tu email real

# Directorios locales
CERT_DIR="$(pwd)/certs"
CREDENTIALS_DIR="$(pwd)/duckdns"
mkdir -p "$CERT_DIR" "$CREDENTIALS_DIR"

# Archivo de credenciales DuckDNS para certbot
CREDENTIALS_FILE="$CREDENTIALS_DIR/duckdns.ini"
echo "dns_duckdns_token = $TOKEN" > "$CREDENTIALS_FILE"
chmod 600 "$CREDENTIALS_FILE"

# Paso 1: Registrar IP en DuckDNS
echo "→ Registrando $FULL_DOMAIN -> $IP en DuckDNS..."
curl -s "https://www.duckdns.org/update?domains=$DOMAIN&token=$TOKEN&ip=$IP" && echo "✔ Hecho"

# Paso 2: Esperar propagación DNS
echo "→ Esperando propagación DNS..."
sleep 30

# Paso 3: Ejecutar certbot con el plugin DuckDNS usando podman
echo "→ Solicitando certificado SSL con Let's Encrypt (usando Podman)..."
podman run --rm -it \
  -v "$CERT_DIR:/etc/letsencrypt" \
  -v "$CREDENTIALS_DIR:/duckdns" \
  docker.io/deliverous/certbot-dns-duckdns \
  certonly \
  --dns-duckdns \
  --dns-duckdns-credentials /duckdns/duckdns.ini \
  --dns-duckdns-propagation-seconds 30 \
  -d "$FULL_DOMAIN" \
  --non-interactive --agree-tos -m "$EMAIL"

echo ""
echo "✅ Certificado generado para: https://$FULL_DOMAIN"
echo "📁 Certificados guardados en: $CERT_DIR/live/$FULL_DOMAIN/"
echo "🔑 Archivos generados:"
ls -l "$CERT_DIR/live/$FULL_DOMAIN/"


```
3. Los certificados los encontraras en certs/live/ibmconcert.duckdns.org/

📁 Certificados generados
Después de ejecutar el script, vas a tener:

cert.pem – Certificado SSL

privkey.pem – Clave privada

fullchain.pem – Certificado + cadena intermedia

4. Refer to the following steps to use your own SSL certificate.
Create a subdirectory folder external-tls in the ibm-concert-std/localstorage/volumes/infra/tls directory.
Name your certificate file as tls.crt and key file as tls.key.
Insert the certificate file (tls.crt) and the key file (tls.key) in ibm-concert-std/localstorage/volumes/infra/tls/external-tls directory.
Run ibm-concert-std/bin/start_service ibm-roja-portal-gw to apply the certificate.

```
cd ibm-concert-std/localstorage/volumes/infra/tls
mkdir external-tls
cp /home/itzuser/certs/live/ibmconcert.duckdns.org/cert.pem tls.crt
cp /home/itzuser/certs/live/ibmconcert.duckdns.org/privkey.pem tls.key


```
